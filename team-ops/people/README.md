# People

**One file per person. The single most valuable folder in this system.**

## Who lives here

- Direct reports — always
- Cross-functional partners you interact with weekly — always
- Peers / leadership / external stakeholders — when there's enough signal to be worth tracking

If you haven't talked to or about someone in 90 days, archive their page.

## File naming

`firstname-lastname.md` (kebab-case). Use `firstname-l.md` only if uniqueness allows. Be consistent — Claude indexes by filename.

## What goes in each file

Use `templates/person-page.md`. The structure is:

- **Snapshot** — who they are, current state
- **What they own** — responsibilities/engagements
- **Strengths / growth edges** — your honest read with evidence
- **Career goals** — in their words, dated
- **Active commitments** — what's owed in both directions
- **Signal log** — append-only, dated entries (the gold)
- **1:1 history** — pointers to meeting notes
- **Feedback log** — given and received

## How signals get added

The Signal log is the heartbeat. Entries arrive two ways:

1. **Auto** — when Claude processes an inbox item that mentions them, an entry gets appended with date, source, and interpretation.
2. **Manual** — you observe something and note it directly.

## Privacy

Contents of this folder are **gitignored**. Never committed.

This is sensitive data — performance observations, candid reads, growth conversations. Treat the same way you'd treat hand-written notes in a locked drawer.
