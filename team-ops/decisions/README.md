# Decisions

**Decision log — what was decided, why, by whom, when.**

## File layout

One file per quarter: `YYYY-Q<n>.md`. E.g., `2026-Q2.md`.

Each decision is a section using `templates/decision-record.md` with a stable ID:

```
DEC-2026-04-29-pricing-tier-rework
```

The ID is reference-able from people pages, engagement pages, and meeting notes.

## What counts as "a decision worth logging"

- Anything **one-way** (hard to reverse) — always log
- Anything **affecting more than one person's work** — log
- Anything you'd want to remember in 6 months
- Anything where you might be asked "why did we do that?"

Not every choice. Not "I picked the blue button." Decisions, not preferences.

## Why this matters most for managers

Most management failures aren't bad decisions — they're undocumented decisions that get re-litigated, forgotten, or quietly reversed. This folder is the cure.

## Privacy

Contents of this folder are **gitignored**. Never committed.
