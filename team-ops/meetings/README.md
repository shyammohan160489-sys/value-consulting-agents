# Meetings

**Processed meeting notes — the long-term record.**

After Claude processes an inbox item, the cleaned-up structured output lands here.

## Folder layout

```
meetings/
├── 2026/
│   ├── 04-29-1on1-mayur.md
│   ├── 04-29-team-standup.md
│   └── 04-30-cust-XYZ-discovery.md
├── 2025/
│   └── ...
```

Year folders. Files named `MM-DD-<slug>.md`.

## What's in each file

Output of `templates/meeting-capture.md`:
- TL;DR + context
- Decisions
- Actions
- People signals
- Engagement updates
- Risks / open questions
- Quotes worth keeping

## Why we keep these (vs. just the routed pieces)

The routed pieces (decisions, actions, signals) are the *atomic facts*. The meeting note is the *narrative* — useful when you need to reconstruct *why* a decision happened or *how* a conversation actually went.

## Privacy

Contents of this folder are **gitignored**. Never committed.
