# Archive

**Closed engagements, departed people, dormant areas, completed actions.**

Don't delete things — archive them. The point of a second brain is being able to reach back.

## Layout

```
archive/
├── engagements/      # Closed engagement pages
├── people/           # Departed team members / partners no longer relevant
├── meetings/         # Older than ~12 months (optional — meetings/ can also stay flat)
├── actions/          # Completed actions older than ~3 months
└── areas/            # Areas no longer being actively managed
```

## When to archive

- **Engagement** — closed (won/lost/delivered) and 30 days have passed → archive
- **Person** — left the team / org → archive immediately (their signal log stops being relevant for live decisions but stays for reference)
- **Area** — no longer something you're actively steering → archive
- **Action** — completed → flow to `actions/done.md`, then to `archive/actions/` after ~90 days

## Privacy

Contents of this folder are **gitignored**. Never committed.
